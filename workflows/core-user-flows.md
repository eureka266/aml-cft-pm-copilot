# Core User Workflows

## 工作流 1: 地址筛选（Address Screening）

### 触发事件
合规团队需要检查某地址是否在黑名单上

### 主要步骤

1. **输入地址** → 用户提交单个或批量地址
2. **系统查询** → Phalcon 检查 AML 黑名单、制裁名单、PEP 列表
3. **返回结果** → 返回 "Match Found" / "No Match" / "Uncertain"
4. **人工审核** → 如有匹配，合规官员人工审查
5. **文档记录** → 审查结果和决定被记录
6. **可能提交 SAR** → 如果确认风险，进入 SAR 生成流程

### 关键决策点

- 如果是模糊匹配（名字接近但不完全相同）→ 标记为待审核
- 如果在多个黑名单上 → 升级优先级

### 保护措施

- ✅ 必须记录所有查询（审计日志）
- ✅ 绝不自动阻止（必须人工审核）
- ✅ 结果必须可追溯

### 相关 UX 模块

- 搜索框（单个地址）
- 批上传（CSV）
- 结果展示表格
- 详情查看面板

## 工作流 2: 交易监控（Transaction Monitoring）

### 触发事件
客户配置监控规则后，交易需要持续评估

### 主要步骤

1. **配置规则** → 管理员设定监控规则（如 > $1M）
2. **实时评估** → 新交易进入系统后立即评估
3. **生成警报** → 匹配规则的交易生成警报
4. **通知团队** → 合规团队收到通知
5. **人工分类** → 合规官员审查和分类警报
6. **存档或升级** → 关闭或转入案例管理

### 关键决策点

- 什么样的交易应该触发警报？
- 警报的优先级如何定义？

### 保护措施

- ✅ 警报必须有溯源（为什么触发）
- ✅ 规则修改必须有审计日志

### 相关 UX 模块

- 规则配置面板
- 警报列表/看板
- 警报详情视图

## 工作流 3: 案例管理（Case Management）

### 触发事件
多个警报需要分组，进入深入调查流程

### 主要步骤

1. **创建案例** → 从警报创建案例或手动创建
2. **添加上下文** → 添加调查笔记、证据、相关警报
3. **分配所有者** → 分配给调查人员
4. **进行调查** → 人工调查，可能需要链上查询或外部信息
5. **决策** → 关闭案例 / 提交 SAR / 升级处理
6. **存档** → 案例记录和决定被永久存档

### 关键决策点

- 什么时候从警报升级到案例？
- 调查人员权限如何管理？

### 保护措施

- ✅ 所有修改都有时间戳和用户记录
- ✅ 关键决定必须有批准

---

## 工作流 4: SAR/STR 报告生成

### 触发事件
调查完成，合规官员需要生成可提交的 SAR/STR

### 主要步骤

1. **从案例生成草稿** → Phalcon 基于案例数据生成 SAR/STR 草稿
2. **编辑和验证** → 合规官员补充信息、验证准确性
3. **获批和签署** → 必要的签署和批准
4. **提交** → 提交至相关监管部门（通常需要外部集成）
5. **存档** → 保留一份副本用于审计

### 关键决策点

- SAR/STR 生成的最少信息是什么？
- 哪些信息需要人工补充？

### 保护措施

- ✅ 绝不自动提交（必须人工批准和签署）
- ✅ 每份报告都有版本历史
