# 数据管理页面

# 背景和目标

在 Pricing Plan V2 调整中，平台对以下资源新增了用量限制：

*   **Labels**（用户为地址/交易自定义的标签）
    
*   **Tags**（用户自定义的通用标签，可应用于多个地址）
    
*   **Monitorings**（地址/交易监控数量上限）
    

为确保用户能够清晰理解资源使用情况，并便于管理其配额，平台需要提供一个统一的资源管理界面，使用户可以：

1.  查看其在当前 Project / Workspace 下已创建的 Labels / Tags / Monitorings
    
2.  进行管理操作：编辑、删除、搜索、筛选等
    
3.  了解当前 Plan 额度不足时如何升级，促进付费转化
    

因此，本次需求将新增一个**统一的资源查看与管理入口：Data Management 页面**，用于整合并管理用户资源。

# 详细方案

新增页面：Data Management

*   Data Management 页面与 Project 绑定。
    
*   页面入口可放置和 Project 比较关联的位置
    

**Data Management 下包含 3 个子模块：**

1.  **Labels**
    
2.  **Tags**
    
3.  **Monitorings**
    

每个模块的界面结构保持一致（列表 + Search + Filter + 分页）。

## Labels

#### Filter 操作

*   Chain
    
*   Type：Address / Transaction
    

支持多选

#### Search 操作

*   地址/交易哈希（Target）
    

*   Label 文本内容
    

模糊匹配

| 字段 | 说明 |
| --- | --- |
| **Target** | 链图标 + 地址/交易哈希（可点击跳详情页） |
| **Type** | Address / Transaction |
| **Label** | 自定义标签内容（单行展示，过长截断） |
| **Action** | Edit / Delete |

*   Edit：弹窗修改标签内容
    

*   Delete：二次确认后删除
    

*   地址点击跳转到对应详情页（平台内部页）
    

## Tags

#### Search 操作

*   Tag Name
    
*   地址
    

| 字段 | 说明 |
| --- | --- |
| **Tag Name** | 标签名称 |
| **Usage Count** | 使用数量；展示如：`24 addresses` |
| **Action** | View Detail / Edit / Delete |

*   View Detail: 
    
    *   点击后出现弹窗 “Tag Usage”，展示所有使用该 Tag 的地址，表格字段：
        
        *   Address： Chain 图标 + Address （可跳转至详情页）
            
        *   Risk Level：地址风险等级
            
        *   Applied At：Tag 应用时间
            
*   Edit：弹窗修改 Tag  内容
    

*   Delete：二次确认后删除
