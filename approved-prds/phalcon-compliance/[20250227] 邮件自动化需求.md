# 一、背景与目标

Phalcon Compliance 目前主要靠邮箱注册，但用户注册后行为断层：注册后未完成首次扫描，扫描过的未持续用，有付费意向的没及时跟进，历史扫描地址有风险变化时也没提醒。  
希望建立一套自动发邮件的机制，根据用户的不同行为触发相应邮件，目标是：

1.  让更多注册用户完成首次扫描
    
2.  让完成首次扫描的用户更频繁使用
    
3.  让有付费意愿的用户更易转化
    
4.  风险变化时，主动提醒用户重新扫描
    

相比过往的产品通知邮件和手动发送的邮件，希望支持：

*   邮件根据用户行为自动触发
    
*   发件人邮箱以真人身份，用户可直接对邮件进行回复
    
    *   售前邮件发件人：ruby.xu@blocksec.com, 自动转发到邮件组sales.blocksec.com
        
    *   售后邮件发件人： jenna.cheng@blocksec.com，自动转发到邮件组phalcon\_support@blocksec.com
        

邮件组的成员回复时，可以用自己的常用邮箱登录，然后配置send email as - add another email address, 填写希望发邮件的身份，比如jenna.cheng@blocksec.com

配置好后，在发邮件时即可在From下拉栏中选择需要的发件人身份

运营可在此配置邮件自动发送规则，系统根据用户行为或定时计划自动触发，无需人工干预。每条规则独立管理触发条件、用户过滤、发送序列和频控设置。

*   **触发条件**：支持行为事件（`user_registered` / `login` / `scan_completed` / `checkout_page_entered` / `payment_succeeded` / `subscription_cancelled`）和定时触发（每月/每周固定时间），可叠加发送延迟
    
*   **用户过滤**：在触发事件基础上，通过 AND 条件组合过滤目标用户，可用字段包括 `scan_count` / `last_active_time` / `last_login` / `plan` / `unsubscribe_status` / `registered_time`；系统自动排除已退订该邮件类型的用户，无需手动配置
    
*   **序列邮件**：一条规则可绑定多个模板，按顺序发送，每封之间可设置等待时长（分钟/小时/天）
    
*   **频控**：每条规则独立配置冷却天数（默认 3 天），并可设置是否允许对同一用户重复触发
    
*   **状态管理**：规则支持草稿 / 启用 / 停用三种状态，可随时切换，不影响已发送数据

### 3.手动群发

运营主动发起的一次性邮件推送，适用于产品更新、法规通报等非自动触发场景，按五步流程完成配置后立即或定时发送，发送记录自动进入追踪与统计。

*   **模板选择**：从已启用的模板库中选择，草稿和停用模板不可选
    
*   **收件人**：支持三种方式指定——从现有分群选择、按用户属性临时筛选（`plan` / `scan_count` / `last_login` / `last_active_time` / `unsubscribe_status` / `registered_time`，仅本次有效不保存为分群）、上传 CSV 或直接粘贴邮箱列表；已退订该邮件类型的用户自动排除
    
*   **预览**：发送前可按语言版本预览邮件渲染效果，核对发件人、受众范围和预计人数
    
*   **发送时间**：支持立即发送或指定日期时间定时发送
    
*   **任务命名**：每次发送须填写任务名称，用于在追踪与统计页识别对应记录

### 4. 追踪与统计

*   **基础指标：** 记录每封邮件的 **Sent** (发送量) 和 **Delivered** (到达量)。
    

1.  **Open Rate (打开率)** - 判断标题/发件人是否吸引人
    
2.  **Click Rate (点击率)** - 判断 CTA 是否有效
    
3.  **Bounce Rate (退信率)** - 保护发件人信誉
    
4.  **Unsubscribe Rate (取消订阅率)** - 判断是否骚扰用户
    

邮件模版：在邮件CTA 中增加utm标记，在GA中建立看板 [UTM管理表](https://notes.dingtalk.com/cspace/doc?spaceId=4871492109&dentryId=206052749786&orgId=246019097&utm_source=search&utm_medium=search_main&dt_editor_toolbar=true)

需要三个页面：

*   **追踪与统计页：**查看所有邮件发送任务的整体结果，运营可以按时间、发送方式、状态等条件筛选，并快速看到每次邮件的发送量、送达量、打开率、点击率、退信数和退订数。
    

*   **发送记录列表页：**定位具体某一次邮件发送任务，运营可以查看邮件标题、模板、发送对象、发送方式（自动/手动）以及关键效果指标，并点击进入详情页进一步分析。
    

*   **发送详情页：**分析单次邮件发送效果，展示完整发送漏斗（目标人数、已发送、已送达、打开、点击）、UTM 链接及 CTA 点击来源，同时显示被系统跳过的用户数量和失败原因，用于判断邮件效果和定位异常。
