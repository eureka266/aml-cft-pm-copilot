# 埋点方案

# 背景与目标

本次上线后，Phalcon Compliance 平台将全面开放订阅模式：

*   用户可自行注册、登录、使用、订阅
    
*   不同 Plan 的用户对平台功能的使用权限不同
    
*   地址/交易详情页展示优化：风险信息总结及风险概览
    

### 埋点目标

*   跟踪用户核心行为路径（注册 → 登录 → 订阅）
    
*   监控关键功能的使用情况（报告导出、详情页操作）
    
*   支持转化分析、留存分析和用户行为洞察
    

# 埋点范围与事件

## Compliance Landing Page

需统计：

*   UV / PV
    
*   地区分布
    
*   落地页 Heatmap
    
*   Start Free Trial 按钮 / 搜索框点击率
    

以上均可由 Clarity 自动收集，无需额外配置

## 平台

*   记录关键转化操作和路径
    
    *   注册：用户从哪跳转注册页面（直接点击 Start Free Trial 按钮还是搜索？）
        
    *   进入 pricing Plan ：用户从哪个页面跳转 Pricing Plan
        
    *   订阅：进入 Pricing Plan 的哪些用户进行了 subscribe 操作？
        
*   记录重点功能的使用情况
    
    *   导出 Address Report
        
    *   导出 Transaction Report
        
*   limit 弹窗弹出功能
    

基本也都可以用 Clarity 和 GA 的自动的事件收集，要补充下面两条：

| 模块 | 事件ID | 参数 |
| --- | --- | --- |
| 注册&登录 | log\_in | user\_id |
| 地址界面 | export\_address\_report | user\_type |
| 交易界面 | export\_transaction\_report | user\_type |
| Limit 弹窗 | limit\_popup\_show | feature, user\_type |
| 弹窗点击升级 | limit\_popup\_upgrade | feature, user\_type |

传 user Id 到 GA：[https://support.google.com/analytics/answer/9213390?ref\_topic=9228654](https://support.google.com/analytics/answer/9213390?ref_topic=9228654)

user\_type: trial, starter, growth, pro, enterprise\_trial, enterprise\_paid

注：可以分阶段实施，国庆前先接入 Clarity 和 GA，其他事件/功能/user type 相关的可以国庆后再加（要记录的 event  可能会增加）
