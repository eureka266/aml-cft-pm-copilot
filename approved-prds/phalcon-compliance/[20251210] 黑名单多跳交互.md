# \[20251210\] 黑名单多跳交互

# 背景与目标

#### 背景

平台当前提供“黑名单（Blacklist）”功能：

*   用户可将地址加入黑名单，筛查到黑名单地址时自动触发 Critical 风险。
    
*   提供 Blacklist Interaction 规则模板，可识别“与黑名单地址有直接交互（1 hop）”的相关地址。
    

但用户发现许多风险来自 2 跳或更多跳的交互路径，当前会漏掉这类风险。

#### 目标

这次改动需要实现：

*   Blacklist Interaction 规则支持：
    
    *   **direct / indirect / both**
        
    *   保留现有的 direction（incoming/outgoing）与金额（value）配置（支持 percent 吗？）
        
*   引擎支持基于黑名单地址的 **多跳路径识别**
    
    *   能根据用户配置的方向、跳数、金额进行过滤
        
*   default risk engine 中的黑名单相关的规则不改变且兼容本次修改
    
*   给用户发邮件通知关于黑名单检测能力的调整
    

# 详细方案

基于对 default risk engine 的兼容性考虑，本次改动采取：保留 Blacklist Interaction 模板不与 Interaction Risk 模板合并；扩展字段支持多跳。

此方案不会影响 default risk engine 或现有用户定义的规则。

## 规则模板调整

现有 Blacklist Interaction 模板：

| Target Type | Trigger Condition |
| --- | --- |
| Address | When a screened address has directly exposure to address in your blacklist in<br>*   \[方向选择框\] transaction<br>    <br>with value greater than or equal to <br>*   \[金额填写框\] USD |
| Transaction | When a screened transaction has directly exposure to address in your blacklist in<br>*   \[方向选择框\] transaction<br>    <br>with value greater than or equal to <br>*   \[金额填写框\] USD |

**调整如下：**

| Target Type | Trigger Condition |
| --- | --- |
| Address | When a screened address is exposed to addresses in your blacklist<br>in<br>*   \[**方向选择框**\] transactions<br>    <br>through<br>*   \[**交互类型选择框**\] interactions <br>    <br>(optional) and satisfies the following rules:<br>*   When the exposure value is more than \[**min**\] and less than \[**max**\]<br>    <br>*   When the exposure percent is more than \[**min**\] and less than \[**max**\] |
| Transaction | When a screened transaction has directly exposure to address in your blacklist in<br>*   \[**方向选择框**\] transaction<br>    <br>through<br>*   \[**交互类型选择框**\] interactions <br>    <br>(optional) and satisfies the following rules:<br>*   When the exposure value is more than \[**min**\] and less than \[**max**\]<br>    <br>*   When the exposure percent is more than \[**min**\] and less than \[**max**\] |

## 引擎支持

由于用户黑名单地址 **不会写入图数据库**，引擎需要在扫描时：

1.  查询“当前筛查地址和黑名单地址集合”之间的路径
    
2.  根据规则进行过滤，按现有风险结果格式返回

| 默认 Engine 名称 | Address with Direct Blacklist Interaction |
| Target Type | Address |

|  | 内容 |
| --- | --- |
| Name | Transaction with Direct Blacklist Interaction |
| Target Type | Address |
| Trigger Condition | *   方向（ Direction）：<br>    <br>    *   deposit or withdrawal<br>        <br>*   交互类型：<br>    <br>    *   direct<br>        <br>*   金额阈值（Exposure Value）：<br>    <br>    *   \> 1,000 |
