# \[20251111\] 地址/交易列表优化

# 问题与目标

现有的地址/交易列表存在以下问题：

1.  **风险信息展示不足**：在列表页难以直接看到风险详情，需进入详情页才能查看。
    
2.  **字段重复**：地址的 _Label_ 与 _Label 列_ 内容重复，占用空间。
    
3.  **展示优先级不合理**：_Alerts_ 的重要性降低，但仍占据较前位置，可考虑后移。
    
4.  **排序限制**：目前仅支持按 _Last Screening Time_ 排序，无法按其他时间（如导入时间）查看
    

### 优化目标

*   在列表页直接展示核心风险信息，让用户一眼看到重点。
    
*   支持自定义排序和筛选，满足不同使用场景。
    
*   优化字段布局与交互。
    

# 详细方案

## 地址列表

#### 表格操作

1.   新增 **Sort By** 功能，用户可选择表格排序方式。  
    排序选项包括：
    

*   Last Screening Time - Latest to Oldest（最近筛查，默认选项）
    
*   Add Time - Latest to Oldest（最近添加）
    
*   Risk Level - High to Low（风险等级从高到低）
    
    *   再按 last screening time 排
        

1.   增加 **Filter** 操作（都放在 More Filters 中，放在前面）：
    

*   Customer：参考 Label 筛选
    
*   Last Screening Time: Date range 筛选

#### 字段调整

| 字段 | 说明 |
| --- | --- |
| **Address** | *   展示内容：链图标 + 地址。<br>    <br>*   若地址存在 Label，则默认显示 Label。<br>    <br>*   由于 _Label_ 字段已从表格中移除，用户可通过 Hover 交互直接修改 Label（参考 PE 中的交互）。 |
| **Risk Level** | *   地址风险等级，与现有逻辑一致。<br>    <br>*   此字段可与 _Risk Summary_ 合并。 |
| **Risk Summary** | *   展示该地址触发的所有风险 Tag（来源于详情页 Risk Summary）（如果有别的方案也可考虑，我暂时只想出来这个）<br>    <br>*   此列为列表中最重要的内容，占较大的显示空间。 |
| **Unresolved Alerts** | *   展示未解决告警数量 / 告警总数，与当前逻辑一致。 |
| **Tag** | *   系统 Tag + 自定义 Tag，与当前逻辑一致。 |
| **Customer** | *   关联的客户，与当前逻辑一致。 |
| **Last Screening Time** | *   最近一次筛查时间，与当前逻辑一致。 |
| **Added Time** | *   初次导入/添加时间 |
| **Re-screening** | *   展示是否开启自动复筛，与当前逻辑一致。 |

Risk Summary 可占较大空间，参考下方大龙手绘示意图：

## 交易列表

#### 表格操作

同样增加 **Sort By** 功能，排序选项包括：

*   Last Screening Time - Latest to Oldest（最近筛查，默认选项）
    
*   Added Time - Latest to Oldest（最近添加）
    
*   Tx Time - Latest to Oldest（最近交易）
    
*   Risk Level - High to Low（风险等级从高到低）
    
*   Value (USD) - High to Low （金额从高到低）
    

#### 字段调整

修改后的表格字段及对应的顺序见下表：

| 字段 | 说明 |
| --- | --- |
| **Transaction** | *   若为交易（Transaction）：展示链图标 + Tx Hash；<br>    <br>*   若为转账（Transfer）：展示 _Transfer + 序号_（如 “Transfer No.1”）。 |
| **Risk Level** | *   交易或转账的风险等级，与当前逻辑一致。<br>    <br>*   同样可与 _Risk Summary_ 合并。 |
| **Risk Summary** | *   展示交易/转账触发的所有风险 Tag（与详情页一致）。该列为主要信息区域。 |
| **Direction** | *   交易方向，和当前保持一致 |
| **Unresolved Alerts** | *   展示未解决告警数量 / 告警总数，和当前保持一致 |
| **Value (USD)** | *   交易或转账金额，按当前逻辑展示。 |
| **Amount** | *   转账数量，与当前一致。 |
| **Asset** | *   交易涉及的资产，与当前一致。 |
| **From / To** | *   分别展示来源与目标地址。交互与地址列表页保持一致（Label 可 Hover 编辑）。 |
| **Tx Time** | *   交易发生时间，与当前一致。 |
| **Customer** | *   关联客户，与当前一致。 |
| **Label** | *   交易 Label，与当前一致。 |
| **Screening Time** | *   最近一次筛查时间，与当前一致。 |
| **Added Time** | *   添加时间 |

#### 其他交互调整

1.  **Transfer 区分度提升**
    
    *   当交易展开展示 Transfer 时，为 Transfer 部分增加背景底色，以区分主交易与子 Transfer 行。
        
    *   当前在列表滚动至右侧时难以分辨层级，此优化可提升视觉清晰度。

## 其他逻辑

#### Risk Summary 风险 Tag 展示逻辑

*   先按 Engine Type：  
    
    *   Exposure
        
        *   Blacklist
            
            *   Blacklist > Blacklist Interaction
                
        *   Entity Risk
            
        *   Interaction
            
    *   Behavior
        
        *   Transit Address （或交易的 Rapid Transit）
