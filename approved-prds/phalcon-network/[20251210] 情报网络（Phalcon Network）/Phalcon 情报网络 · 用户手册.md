# Phalcon 情报网络 · 用户手册

# 快速概览

**Phalcon Network 是什么？**   Phalcon Network 帮助您实时检测流入您链上地址的非法或高风险资金，并及时通知您。   一旦完成监控配置，您就无需再手动追踪链上动态。系统将持续扫描所有转入交易，并在检测到高置信度风险时，通过您设定的通知渠道发送警报。

**3 步设置，接收风险警报**

1.  **添加监控地址**   输入您想要追踪的钱包地址，我们将持续监控其风险活动。
    
2.  **配置通知渠道**   接入您常用的协作工具（如 Lark），确保警报能实时送达对应团队。
    
3.  **接收并管理风险警报**   查看警报详情，及时响应，并在调查过程中随时跟踪警报状态。
    

完成设置后，一旦有风险资金流入您监控的地址，警报将自动推送。

**我会收到怎样的警报？**   每条警报对应一笔具体的转入交易，包含以下信息：

*   检测到的风险类型
    
*   来源地址 → 您监控的地址
    
*   交易金额与时间戳
    
*   直接跳转至详细风险分析及资金流向可视化的链接
    

**想了解如何设置警报？**   请前往 **基础指南** 部分查看更多信息。

# 2 添加待监控的地址

*   钱包地址或合约地址均可。
    
*   支持多链（覆盖范围与 Phalcon Compliance 一致）。
    
*   目前仅监控 **入金** 方向交易。
    

## 添加步骤

1.  在监控列表页，输入您需要监控的地址。
    
2.  系统将自动识别并展示该地址在各链上的存在，勾选需要监控的链。
    
3.  （**强烈建议**）填写一个便于识别的备注（Note），例如“公司热钱包-主网”。
    
4.  点击保存，监控立即生效。
    

## 告警内容示例

每条告警均对应一笔具体的入金交易，信息清晰完整，便于您快速研判：

*   **风险类型**：明确命中的风险标签。
    
*   **资金流向**：从 **来源地址** 到 **您的监控地址**。
    
*   **交易详情**：涉及的 **金额** 与 **时间**。
    
*   **分析入口**：可直接跳转查看完整风险详情与资金流向的链接。

你需要准备什么：

*   一个 Telegram 账号
    
*   可以与 Bot 对话的权限

# 4.接收告警

如何理解一条告警：

风险类型 & 来源、出金地址 及 入金地址、交易金额 （USD）& 交易时间

你可以在告警详情页直接跳转我们的链上反洗钱合规平台Phalcon Compliance进行进一步调查

**监控规则说明**

*   **阈值触发**：仅当单笔入金金额 **≥ 系统预设风险阈值** 时，才会进行风险判断。
    
*   **风险即告警**：一旦命中高风险资金来源，系统将立即触发一条告警。
    
*   **告警粒度**：单笔交易仅会生成 **1条** 对应告警，避免信息轰炸。

# 6 FAQ

## Phalcon Network 主要解决哪些问题，不解决哪些问题？

Phalcon Network 解决的是“是否存在已知高风险资金流入”的感知问题，帮助用户第一时间获知风险暴露情况。它并不提供完整的合规判断、处置建议、账户冻结决策或监管报告生成等功能，这些需要在更完整的合规分析产品中完成。

## 告警是基于地址、交易，还是资金流生成的？

Phalcon Network 的告警以具体入金交易为触发点，但风险判断基于该交易对应的资金流关系。告警并非单纯针对地址或交易本身，而是反映该笔入金在资金来源维度上的风险暴露情况。

## 为什么没有收到告警？

这可能是因为你的入金交易所涉及的资金没有达到默认的引擎的阈值（这个阈值通常为价值1w美金的虚拟货币）或者你的入金地址不涉及非法的风险标签

## 情报来源是哪些？

目前BlockSec Phalcon Network仅使用：BlockSec 内部高风险地址标签库

这是一套与 Phalcon Compliance 中高风险 Risk Indicator 保持一致的公共情报集合，涵盖：

制裁、恐怖融资、诈骗、勒索软件、洗钱、混币、黑市、暗网业务等。

## 会不会有误报？

情报网络的设计目标是 **“高置信度、低频率”**，而不是覆盖所有可疑行为，因此：

*   只有命中**高风险或非法情报**的入金才会触发告警
    
*   单笔交易只生成 **1 条告警**
    
*   多条风险路径会被合并展示，避免重复轰炸
    

但需要理解的是：

*   情报网络提供的是**风险提示（Signal）**，不是最终定性
    
*   对单笔交易的深入调查，建议跳转至 **Phalcon Compliance** 查看完整风险上下游
    

## 我能否自定义风险类型或阈值?

在Phalcon Network中，风险类型与预警阈值均采用集中管理机制，确保所有用户接收到语义一致、客观中立的预警提示。

若您需要：  
● 自定义风险分类  
● 可调整的预警阈值  
● 更精细的扫描逻辑或自动重复扫描功能

您可在 Phalcon Compliance 中配置这些选项，该平台基于同一数据库下的风险检测引擎，为您提供高级监控、自定义设置及合规工作流支持。

## 我想监控的地址在当前不支持的链上怎么办

Phalcon Network 目前仅支持在已覆盖的区块链上进行地址监控。

如果你的地址所在链暂未支持，你可以通过**phalcon\_support@blocksec.com** 联系我们，说明你的使用场景和需求。我们会持续根据风险覆盖能力和用户需求评估并扩展对更多区块链的支持。  

**暂停监控后，告警会被补发吗？**

当用户暂停某个地址的监控后，该地址在暂停期间产生的入金行为不会触发告警。监控恢复后，系统不会对暂停期间的历史交易进行回溯分析或补发告警。

**Phalcon Network 支持哪些区块链？不支持的链会怎样？**

Phalcon Network 仅对当前已支持的区块链提供监控和告警能力。若被监控地址所在的区块链暂不受支持，系统将无法对其链上活动进行分析，也不会生成相关告警。

**什么时候需要使用 Phalcon Compliance？**

当用户需要对告警进行更深入的调查，例如查看完整资金路径、持续重筛地址风险、使用可配置的风险规则，或将风险分析结果纳入合规流程时，应使用 Phalcon Compliance。Phalcon Compliance 与 Phalcon Network 基于同一风险引擎，但提供更完整的分析与合规能力。

## 与 Phalcon Compliance 的关系

情报网络是 **Phalcon Compliance** 的延伸与入口：

*   免费情报可用于初步风险感知
    
*   如需高精度监控、批量处理与合规报告，可升级至 Phalcon Compliance
    
*   两者共享同一风险引擎与标签体系
